# Rest API Reference

Basestack Feature Flags is designed with a client/server architecture to offer seamless functionality and efficient communication. The core of this architecture lies in the REST API server, which serves as the backbone for handling feature flags across the system.

When you integrate the Basestack SDK into your application, it acts as the client, enabling you to interact with the REST API server effortlessly. This client-server communication empowers you to easily manage feature flags and utilize various functionalities within your application

## API Explorer

These two main endpoints are essential for consuming the SDK aspect of the API.

### Get All Flags

```sh copy
curl "https://your-basestack-hosted-app-domain.com/api/v1/:projectKey/:envKey/flags"
```

### Get Flag by Name

Retrieve a specific feature flag linked to a particular project and environment (e.g., Develop, Staging, etc).

```sh copy
curl "https://your-basestack-hosted-app-domain.com/api/v1/:projectKey/:envKey/flags/:name"
```

#### Typescript Type Definition
